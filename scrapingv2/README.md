## 📂 Directory Structure & File Glossary

### 1. Root Execution & Data Generation (`/`)
These are the primary entry points for generating your raw data and personas.
* **`get_jsons.py`**: A standalone script that safely scrapes Google Scholar (handling pagination to pull 50 papers) and saves the raw data to `data/raw_authors/`.
* **`run_persona_standalone.py`**: The execution script that reads the scraped JSONs, triggers the LLM pipeline, and saves the structured output to `data/personas/`.
* **`requirements.txt`**: Project dependencies.

### 2. Core Application Logic (`app/services/`)
This folder contains the engine that powers the persona generation.
* **`app/services/scraping_service.py`**: The core API integration with SerpApi. Includes zero-shot LLM relevance filtering to drop out-of-domain papers and strict pagination logic to protect API credits.
* **`app/services/persona_pipeline.py`**: The active 4-stage, Pydantic-enforced LLM pipeline. Translates raw academic paper metadata into a behavioral evaluation profile.
* **`app/services/persona_pipeline_old.py`**: Legacy/deprecated version of the pipeline.
* **`app/services/file_store.py`**: Utility service for managing file reads/writes.

### 3. Annotation & Ablation Experiments (`app/batch/`)
These scripts execute the code hunk classification against the ground-truth dataset (`_only_context.json`) and output the predicted results (`_api_results.json`).
* **`app/batch/split_dataset.py`**: Safely splits the full ground truth files into an 80% Training Set (for Few-Shot examples) and a 20% Testing Set (for evaluation).
* **`app/batch/run_full_baseline_batch.py`**: *(Experiment 1)* Annotates the dataset using standard software engineering instructions. No persona, no historical examples.
* **`app/batch/run_annotation_batch.py`**: *(Experiment 2 - The Winner)* Annotates the dataset by injecting the generated academic persona into the system prompt (Zero-Shot Persona).
* **`app/batch/run_fewshot_batch.py`**: *(Experiment 3)* Annotates the test set by injecting up to 30 historical ground-truth examples into the prompt. No persona.
* **`app/batch/run_hybrid_batch.py`**: *(Experiment 4)* Annotates the test set by combining the academic Persona with the historical Few-Shot examples.
* **`app/batch/run_persona_batch.py`**: Legacy script for running the pipeline through the database.
* **`app/batch/run_batch_metrics.py`**: The automated evaluation script. Calculates Cohen's Kappa, F2-Weighted scores, and prints precision/recall reports.

### 4. Database & Models (`app/db/` & `app/models/`)
* Contains legacy SQLAlchemy schemas (`author.py`, `persona.py`, `experiment.py`, `hunk.py`) and database connection logic (`database.py`) initially built for persistent storage. 

### 5. Data Storage (`data/`)
* **`data/raw_authors/`**: Contains the raw, unformatted Google Scholar output JSONs generated by `get_jsons.py`. get_jsons puts the file to the backend/raw_authors. I manually moved it to data/raw_authors.
* **`data/personas/`**: Contains the final, LLM-generated psychological profile JSONs.

The output logs for every stage of the ablation study validate the pipeline's effectiveness. These text files contain the precision, recall, F1, and Cohen's Kappa scores for each run:

* **`results_full_baseline.txt`**: The control group (no persona).
* **`results_handmade.txt`**: The theoretical ceiling using human-crafted profiles.
* **`results_without_weights.txt`**: The winning automated Zero-Shot Persona methodology.
* **`results_with_weights.txt`**: The Zero-Shot Persona utilizing citation-count weighting (resulted in drops due to popularity bias).
* **`results_fewshot.txt`**: Pure historical pattern matching (no persona).
* **`results_scraping_fewshot.txt`**: The Hybrid model (suffered from Context Clash).
* **`results_self_check.txt`**: The Chain-of-Thought architecture (suffered from Sycophancy).
* **`results_hierarchy.txt`**: The Codebook architecture (suffered from probabilistic misfires).
* **`results_cluster.txt`**: The Intent Clustering architecture (suffered from the Black Hole Effect).

## 🚀 Installation & Setup

This project uses a streamlined Python environment. It does not require a database or a local web server to run.

### Prerequisites
* Python 3.10 or higher.
* API Keys for **Gemini** (Google AI Studio) and **SerpApi**.

### Step 1: Clone and Set Up the Virtual Environment
Open your terminal and run the following commands to isolate the project dependencies:

**For Mac/Linux:**
```bash
git clone <your-repository-url>
cd tangled-commits-persona
python3 -m venv venv
source venv/bin/activate
```

**For Windows:**
```bash
git clone <your-repository-url>
cd tangled-commits-persona
python -m venv venv
venv\Scripts\activate
```

### Step 2: Install Dependencies
With your virtual environment active, install the required packages:
```bash
pip install -r requirements.txt
```

### Step 3: Configure Environment Variables
Create a file named `.env` in the root directory of the project and add your API keys. The scripts are configured to automatically load these credentials.

```env
GEMINI_API_KEY=your_gemini_api_key_here
SERPAPI_KEY=your_serpapi_key_here
```
